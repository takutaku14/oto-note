# **アマチュア音楽団体支援アプリ 基本設計書**

## **1\. システムアーキテクチャ・技術スタック**

本システムは、保守性と開発スピードを重視したモダンなSPAとして構築し、将来的なSaaS拡大とCloud Functions導入を見据えた柔軟なアーキテクチャを採用する。

### **1.1. フロントエンド技術**

* React 18+ / Vite / vite-plugin-pwa（Service Workerによるキャッシュおよびプッシュ通知受信用）  
* TypeScript / Tailwind CSS / React Context API ＋ SWR  
* **API通信の抽象化**: 将来のCloud Functions移行に備え、GASなどの外部APIを叩く処理は直接コンポーネントに書かず、必ず lib/api/ ディレクトリ内に集約・隠蔽する。これによりバックエンド移行時の改修範囲を最小化する。

### **1.2. バックエンド・インフラと認証フロー**

* **ログイン・ユーザー管理**: Firebase Authentication  
* **データベース**: Cloud Firestore  
* **プッシュ通知配信基盤**: Firebase Cloud Messaging (FCM)  
* **外部API連携 (初期)**: Google Apps Script (GAS) Web API を活用したサーバーレス連携。  
  * **セキュリティ要件**: POST通信のみ許可、簡易APIキーによる認証、Firebase UIDによる名簿照合（バリデーション）。

## **2\. コア設計思想**

1. **マルチテナントモデル（複数団体対応）**  
2. **全団体横断のパーソナライズUX（マイタイムライン）**  
3. **シーズン（公演・プロジェクト）ベースの情報管理**  
4. **ユニバーサル・イベントモデル**（練習、集金、当番等あらゆる事象を統合）  
5. **イベントにおける「作成者」と「責任者」の分離**  
6. **汎用的かつスケーラブルなデータ構造**（パート、役職バッジ等）

## **3\. 画面一覧・画面遷移設計**

### **3.1. 共通・一般向け画面**

* **ログイン (/login)**  
* **ホーム：マイタイムライン (/home または /)**  
  * ログイン直後のトップ画面。自分が所属する全団体の「自身が対象となっているイベント・タスク」を横断的にフェッチし、時系列の縦スクロールタイムラインとして統合表示する。  
  * 各カードには団体名や団体カラーのバッジを表示し、視認性を高める。  
* **団体選択画面 (/organizations)**：※退団済みの団体は一覧に表示されずアクセス不可。  
* **団体参加画面 (/join/:inviteToken)**：フロントエンドのマスターデータに基づく「セクション → 楽器・役割」の階層型カスケードパート選択UI。  
* **団体別ダッシュボード (/org/:orgId/)**  
  * 選択した団体のみのイベント表示、シーズン選択プルダウン、本番日カウントダウンUI。  
  * ヘッダーに「通知アイコン（未読件数バッジ付き）」を配置。  
* **通知一覧画面 / 通知センター (/notifications)**  
  * 個人宛に届いた通知（「重要なお知らせ」「責任者への任命」など）の履歴一覧。既読/未読ステータスを表示し、タップで対象画面へ遷移し既読化する。  
* **イベント詳細画面 (/org/:orgId/events/:eventId)**  
  * イベントの「責任者（Manager）」の名前を表示。  
  * 各カテゴリに応じたアクションボタン。アクション実行時、オプティミスティックUI（楽観的UI更新）を用いて即座に画面ステータスを切り替える。  
  * **完了UI**: 集金イベントにおいて、自分のステータスが「支払報告済」かつ「受領確認済（幹部承認済）」の場合、安心感を与える完了UI（緑色のチェックマーク等）を表示する。  
* **掲示板一覧/詳細 (/org/:orgId/boards, /org/:orgId/boards/:boardId)**  
  * 選択中のシーズンに関連するスレッドのみを表示。  
  * **【NEW】** 掲示板一覧は、自分が targetUserIds に含まれているスレッド（または全員公開スレッド）のみが表示される。  
  * スレッド内のコメント投稿時、Google Drive等の外部ストレージへの連携リンクを添付・発行する機能。  
* **メンバー名簿 (/org/:orgId/members)**：役職バッジ表示。※退団者は非表示。  
* **マイページ (/mypage)**：プッシュ通知のON/OFF設定。

### **3.2. 幹部および「イベント責任者」専用画面・機能**

* **団体・シーズン設定画面 (/org/:orgId/admin/settings)**（幹部のみ）  
  * シーズンの作成、本番日の設定。  
  * Google Driveやスプレッドシート、カレンダーなどの外部API連携設定（OAuth認証によるトークン取得と保存）。  
* **イベント/掲示板スレッドの作成・編集 (/org/:orgId/admin/events/new 等)**  
  * **対象者（公開範囲）の一括選択UI**: セクションボタンやバッジによるショートカット選択（イベントと掲示板で共通のUIコンポーネントを使用）。※休団者はデフォルト選択から外す配慮を行う。  
  * **責任者（Manager）選択UI**: デフォルトは操作ユーザー自身だが、名簿から任意のメンバーを責任者として指定・委譲可能。  
* **イベント管理・回答一覧画面 (/org/:orgId/admin/events/:eventId/responses)**（幹部または責任者のみ）  
  * イベントの管理者ビュー。集金イベントにおいて、支払報告が上がってきたメンバーに対して\*\*「受領確認済」\*\*ボタンを押してステータスを確定させる（双方向確認の完了）。  
* **メンバー管理機能 (/org/:orgId/admin/members)**（幹部のみ）  
  * ロール変更、自由記述による「役職バッジ」の追加・削除機能。  
  * ステータス（現役/休団/退団）の変更機能。※退団者も含めた全メンバーの閲覧・管理。

## **5\. データモデル設計（Firestore論理スキーマ）**

*※マイタイムラインの実現には、所属する全団体のイベントをパラレルでフェッチしてフロント側でマージ（ソート）する手法をとる。*

### **コレクション: users**

* uid (String, PK), email, displayName, currentOrgId  
* **fcmTokens (Array\<String\>)**: プッシュ通知送信用のデバイストークン配列（複数端末ログイン対応）

### **コレクション: organizations（団体マスタ）**

* id (String, PK), name, inviteToken, createdAt, currentSeasonId  
* **googleAuthTokens (Map)**: OAuth連携で取得したカレンダーやDrive操作用のAPIトークン情報

### **コレクション: seasons（サブコレクション）**

* id (String, PK), title (String), concertDate (Timestamp), createdAt

### **コレクション: memberships（サブコレクション）**

* userId (String, PK & FK), role (String), part (String), section (String), badges (Array\<String\>)  
* **status (String)**: active (現役) | inactive (休団) | withdrawn (退団)

### **コレクション: notifications（memberships 内サブコレクション）**

メンバー個人の通知履歴（インボックス）を管理する。

* id (String, PK)  
* type (String): important\_notice | manager\_assigned 等  
* title (String), body (String), link (String)  
* isRead (Boolean): 既読フラグ  
* createdAt (Timestamp)

### **コレクション: events（サブコレクション）**

* id (String, PK), seasonId (String, FK)  
* category (String): practice | section | billing | survey | notice | duty | return  
* title, memo  
* **targetUserIds (Array\<String\>)**: このイベントの対象となるメンバーのUID配列  
* **authorId (String)**: 作成者のUID  
* **managerId (String)**: 責任者のUID（ステータス管理等の権限を持つ）  
* **（カテゴリ別固有フィールド）**: dueDate (期限), amount (金額), formUrl (アンケートURL), timetable (スケジュール配列), date, externalCalendarEventId

### **コレクション: responses（イベント内サブコレクション）**

イベントに対するメンバーの回答アクションを保持する。

* id (String, PK) \= userId  
* **status (String)**:  
  * 共通: unanswered (未回答) / confirmed (確認済)  
  * 出欠: present (出席) / absent (欠席) / pending (保留)  
  * 集金: unpaid (未払い) / paid\_reported (支払報告済) / **receipt\_verified (受領確認済)**  
* detail (String): 欠席理由などの付加情報  
* updatedAt (Timestamp)

### **コレクション: boards（サブコレクション）**

* id (String, PK), seasonId (String, FK), category, title, body, authorId, createdAt  
* **targetUserIds (Array\<String\>)**: 【NEW】このスレッドを閲覧可能なメンバーのUID配列（空の場合は全員公開など柔軟に設計）  
* **externalLinks (Array\<String\>)**: Google Drive等への共有資料リンクを保持
